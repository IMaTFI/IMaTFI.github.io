最小生成树的形成

假定有一无向图和权重函数, 我们希望找出图的一颗最小生成树。

我们使用贪心算法来解决这个问题。贪心算法管理一个遵守下述循环不变式的边集和: 每轮循环之前, 是最小生成树的一个子集。在当轮循环，我们要做的就是选择一条边，将其加入子集中，使仍然为最小生成树的一个子集，我们称这样的边为安全边。

找出最小生成树的关键在于如何选择安全边。在介绍具体的方法之前，我们先引入一些定义。无向图的一个切割是对集合的一个划分。如果一条边的一个端点位于集合，令一个端点位于集合，则称这条边横跨切割。如果集合中不存在横跨切割的边，则称该切割尊重集合。在横跨切割的所有边中，称权重最小的边为轻量级边。

随着最小生成树算法的推进，总保持着无环的状态。在算法执行的任意时刻，图是一个森林，图中的每一个连通分量是一颗树。所有对的安全边所连接的是中不同的连通分量。

Kruskal算法

Kruskal找到安全边的方法是，在所有连接森林中不同的两棵树的边中，找到权重最小的一个边，时间复杂度为。伪代码如下：

MST-KRUSKAL(G，w)
    A = 空集
    for each vertex v in G.V
        MAKE-SET(v)
    sort the edges of G.E into nondecreasing order by weight w
    for each edge(u,v) in G.E, taken in nondecreasing order by weight
        if FIND-SET(u) != FIND-SET(v)
            A = A∪{(u,v)}
            UNION(u,v)
    return A

FIND-SET(u)用来返回集合的代表元素，UNION用来合并两个集合。

Prim算法

Prim算法中的集合总是构成一棵树，这棵树从一个任意的根节点开始，一直长大到覆盖中的所有结点为止。算法每一步在连接集合和之外结点的边中，选择一条轻量级边加入中。伪代码如下：

MST-PRIM(G,w,r)
    for each u in G.V
        u.key = ∞
        u.pi = NIL
    r.key = 0
    Q = G.V
    while Q != 空集
        u = EXTRACT-MIN(Q)
        for each v in G.Adj[u]
        if v in Q and w(u,v) < v.key
            v. pi = u
            v.key = w(u,v)

在算法的执行过程中，所有不在中的结点都存放在一个基于属性的最小优先队列中中。对每个结点，属性保存的是连接树中结点的所有边中的最小边的权重。
