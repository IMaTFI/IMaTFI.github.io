Learning Relaxed Belady for Content Distribution Network Caching学习笔记

Posted on 2022-07-23 Edited on 2022-07-28 In lab

参考资料：

  • Learning Relaxed Belady for Content Distribution Network Caching

Abstract

CDN：Content Distribution Network
LRB：learning Relaxed Belady
WAN：Wide Area Network

1 Introduction

2022年，CDN在网络传输中的占比预计高达72%

CDN的miss会带来额外成本，为此要设法降低byte miss ratio。

目前CDN使用的最先进的cache算法是启发式的LRU算法，该算法在某些情景下表现较好，但在其他情境下表现较差。

尽管关于cache的研究已经进行了50年，但在byte miss ratio方面，online算法和offline算法之间仍有很大的差距。

Belady算法旨在淘汰距离下一访问最远的object，而Relaxed Belady则随机淘汰距离下一次访问超过一定界限的object。本文中，作者使用机器学习的方法对Relaxed Belady进行了近似。

何为heuristics？

Belady boundary：被Belady算法淘汰的object中，距离下一次访问最近的时间。将Belady boundary作为Relaxed Belady中的界，可以很好的近似Belady算法。

byte miss ratio是一种end-to-end的评价标准，这意味着对于一种配置，要经过几个小时的长期跟踪才能对其进行评判。

good decision ratio：淘汰的object中，距离下一次访问超过界的比例。作者以good decision ratio为最终评价标准，对机器学习模型进行训练。

2 Background and Motivation

2.1 The Challenge of Reducing WAN Traffic

寻找方法去降低CDN的byte miss ratio

CDN cache的容量很紧张，object之间的竞争很激烈

大约有75%的object被访问后的两天内都不会再被访问，称为one-hit-wonder。让这样的object进入cache无疑会造成空间浪费。CDN的cache算法提升空间很大。

CDN的workloads随地理位置、时间而显著变化，所以需要能随时间和访问模式而变化的cache算法。

2.2 Opportunity and Requirements


尽管CDN cache的空间很紧张，但CPU算力很充足，可以额外容纳一些复杂的cache算法。

三个限制条件：

  1. 存储空间只有几GB
  2. 没有TPU和GPU
  3. 每秒处理数万个请求

3 Approximating Belady Algorithm

3.1 Relaxed Belady Algorithm

机器学习算法很难近似Belady算法。首先，计算每一个object的下一次访问时间的成本过高；其次，精确预测object的下一次访问时间很难。

Relaxed Belady algorithm根据到下一次访问的距离是否超过一定的界限，将cache中的object划分两部分A、B。若A非空，则从A中随机淘汰；若A为空，则对B执行Belady算法。

Relaxed Belady algorithm有两个重要性质：

  1. 当界限设得很大时，算法的byte miss ratio和Belady算法很接近。
  2. 每次淘汰时有很多选择。

基于上述两条性质，ML算法的时间消耗和空间消耗能显著减少，因为ML算法只需要在objects的一个小子集中进行选择即可，只要这个集合中有符合淘汰条件的object。ML算法的精度也不需要很高，实质上只需要判断object的下一次访问时间是否越界即可。

找到合适的界限很重要。

3.2 Belady Boundary

Belady boundary：被Belady算法淘汰的object中，到下一次访问距离最近的时间。

Belady boundary的计算需要未来的知识，实际使用中，只需根据训练集设定界限即可。

由表2可以看出，relaxed Belady相比于Belady的差距是可接受的。

3.3 Good Decision Ratio

byte miss ratio可以从总体上评价决策的好坏，但不能知道哪些是好决策、哪些是坏决策。

good decision ratio：淘汰的object中，到下一次访问的距离超过界的比例。

4 Design of Learning Relaxed Belady Cache

ML算法中的4个关键方面：

  1. 决定多少来自过去的信息是可用的。
  2. 如何根据过去的信息动态产生由标签的训练集。
  3. 选择好的ML模型，进行特征选择、目标预测、损失计算、淘汰决策。
  4. 如何选择候选者。

ML算法的目的是提高good decision ratio

4.1 Past information

LRB只保存最近一次request出现在sliding memory window内的object的信息，并用sliding memory window（后称滑动窗口）内的信息进行训练和预测。

这并不意味着没在滑动窗口内的object没有特征值。要注意past information和training data的区别。但不在窗口内的object的信息比较少，只有静态信息和距离上一次访问的距离。

LRB通过验证集选择合适的滑动窗口大小。对于小cache，验证集可以达到最优化滑动窗口的效果，使其能最大化good decision ratio；对于大cache，使用最小二乘法拟合出的直线来选取窗口大小。

4.2 Training Data

构建训练数据是有挑战性的：特征随时间不断变化，且label直到下一次request出现前都不能确定。因此，LRU先产生没有标签的数据。

LRB每隔一段时间便从滑动窗口中选取一些object，记录下这些object的特征。作者之所以不根据request选取，或者只从cache中的object中选取，是为了避免对popular object的偏好。

LRB用两种方法标记数据：

  1. 等到该object下一次被request。这种方法存在明显的缺陷，比如这种方法永远不能得到只被request一次的object的标签，而这样的object恰恰是需要终点学习的。
  2. 当该object距离下一次被request的时间已经超过了Belady boundary，就将其标记。LRU使用这种方法标记滑动窗口以外的object。

4.3 ML Architecture

4.3.1 Features：Deltas，EDCs，and Static

Deltas：代表上一次request到现在的距离，前第n次request到前第n-1次request的距离。如果该object只被request了n次，则。LRU实质上就是根据来淘汰object。

EDCs：exponentially decayed counters。每个EDC均初始化为0。每当一个request到来时，更新。只有在该object的request到来时才更新，而Deltas则是每当request到来时就更新。观察上述公式可知，越小，衰减越快。所以如果有一个object，在过去十分流行，而最近不流行，则其相对较小，相对较大。

Static features：object的大小、类型等。这些特征容易得到且内存消耗很小。

作者为object设置了32个Deltas和10个EDCs

4.3.2 Model：Gradient Boosting Machines

4.3.3 Prediction Target：log（time-to-next-request）

LRB的预测值为到下一次request的距离。LRB实际上只对sliding memory window内的object进行预测，其余object的预测值直接设置为window大小的2倍。

4.3.5 Training data Size：128K

每当训练数据达到128K时，LRB就训练一个新的模型，然后清空训练数据。

4.4 Eviction Candidate Selection

LRB随机从cache中的objects选择64个对其进行预测，淘汰其中预测值最大者。这个过程只需要30微妙。

这可能就是回归模型比二分类模型更优越的原因，回归模型能淘汰越界object中更值得淘汰的那一个。

4.5 Putting All Together


如上图所示，LRB根据长度和Belady boundary相似的滑动窗口内的objects进行学习。这些object的特征被保存在一个紧凑的数据结构中，当新的请求到来时，更新object的特征，同时移动滑动窗口。LRB持续地从上述的数据结构中对数据进行采样，当训练数据集达到128K时，LRB开始训练GBM模型，同时清空训练数据集。当训练数据集再次达到128K时，LRB便开始训练新模型。如果当前的request触发了cache miss，LRB随机从cache中选取64个object并使用GBM模型对其进行预测，淘汰距离下一次访问的预测值的最大者。

8 Conclusions

在这篇论文中，作者给出了LRB算法的设计、应用和评估，展示了将机器学习算法应用于CDN cache的可行性。使用机器学习算法去近似Belady算法的好处在于它可以根据不同的request模式来给出不同的淘汰选择。

更重要的是，作者引入了relaxed Belady算法、Belady boundary和good decision ratio作为淘汰算法的评价标准。

LRB不需要GPU和加速器，且对CPU和内存的消耗是很小的。

但是LRB算法和relaxed Belady算法之间仍有差距，这种差距或许可以通过提高LRB预测的准确率来逐渐弥补。
