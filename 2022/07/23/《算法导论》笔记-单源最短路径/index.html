最短路问题的最优子结构

运用“剪切-粘贴”技术。已知为结点到结点的最短路径。假设中的不是结点到结点的最短路径，则一定有：，这与是结点到结点的最短路径相矛盾！

证明了最短路问题的最优子结构，我们便可以尝试利用动态规划和贪心算法求解问题。

松弛操作

对于每一个结点，我们维持属性，分别表示表示源点到结点的最短距离的上界，和结点的前驱节点。
对边进行松弛操作的过程为：

RELAX(u, v, w)
    if v.d > u.d + w(u, v)
        v.d = u.d + w(u, v)
        v.pi = u

初始化

INITIALIZE-SINGEL-SOURSE(G,s)
    for each vertex v in G.V
        v.d = inf
        v.pi = -1
    s.d = 0

Bellman-Ford算法

Bellman-Ford可以处理存在负权边的最短路问题。

BELLMAN-FORD(G,w,s)
    INITIALIZE-SINGLE-SOURCE(G,s)
    for i = 1 to |G.V|-1
        for each (u,v) in G.E
            RELAX(u,v,w)
    for each (u,v) in G.E
        if v.d > u.d+w(u,v)
            return FALSE
    return TRUE

Dijkstra算法

Dijkstra算法要求所有边的权值非负。

DIJKSTRA(G,w,s)
    INITIALIZE-SINGLE-SOURCE(G,s)
    s = 空集
    Q = G.V
    while Q != 空集
        u = EXTRACT-MIN(Q)
        S=S∪{u}
        for each vertex v in G.Adj[u]
            RELAX(u,v,w)

直白地说，Dijkstra算法在执行过程中维持一个点集，用以保存最短路径已经被找到的点。算法重复从中找到值最小的点，将其加入集合中，然后对所有出发的边进行松弛。
