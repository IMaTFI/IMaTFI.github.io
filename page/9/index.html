题目链接 L2-004 这是二叉搜索树吗？ (25 分)

问题描述

一棵二叉搜索树可被递归地定义为具有下列性质的二叉树：对于任一结点:

  • 其左子树中所有结点的键值小于该结点的键值；
  • 其右子树中所有结点的键值大于等于该结点的键值；
  • 其左右子树都是二叉搜索树。

所谓二叉搜索树的“镜像”，即将所有结点的左右子树对换位置后所得到的树。

给定一个整数键值序列，现请你编写程序，判断这是否是对一棵二叉搜索树或其镜像进行前序遍历的结果

输入

输入的第一行给出正整数 N（≤1000）。随后一行给出 N 个整数键值，其间以空格分隔。

输出格式

如果输入序列是对一棵二叉搜索树或其镜像进行前序遍历的结果，则首先在一行中输出YES，然后在下一行输出该树后序遍历的结果。数字间有1个空格，一行的首尾不得有多余空格。若答案是否，则输出NO。

分析

  • 根据前序遍历的特点, 对于某一结点而言, 先输出当前结点的键值, 再前序遍历输出左子树的键值, 最后前序遍历右子树的键值.
  • 假设我们得到了当前结点的前序遍历序列a[root], a[root+1], a[root+2], ... , a[tail]. 根据二叉搜索树的性质, 我们可以知道:
    1. a[root+1] -> a[tail]中, 第一个大于等于a[0]的 (假设其为a[i]), 一定是a[0]的右孩子
    2. a[tail] -> a[root+1]中, 第一个小于a[0]的 (假设其为a[j]), 一定是a[0]的左子树中的最后一个结点, 即输出a[j]后, 紧接着就要输出a[i]
  • 由此我们可以知道, 假如当前序列为二叉搜索树的前序遍历序列, 则对每一个结点, 一定满足i-j == 1
  • 我们还可以知道, a[root+1], ... , a[j] 为当前结点左子树的前序序列, a[i], ... , a[tail]为当前结点的右子树的前序序列.

Posted on 2022-07-23 Edited on 2022-07-29 In algorithm

例1 P1379 八数码难题

算法思想:

  • 假设当前棋盘状态用一字符串a表示, 根据题目要求, 0有四种移动方式(假设0的位置为x):
    1. 向上移动: swap(a[x], a[x-3])
    2. 向下移动: swap(a[x], a[x+3])
    3. 向左移动: swap(a[x], a[x-1])
    4. 向右移动: swap(a[x], a[x+1])
  • 我们注意到, 这四种移动方式,都对0的位置有一定的要求:
    1. 向上移动: 0不能在第一行, 即x/3 > 0
    2. 向下移动: 0不能在第三行, 即x/3 < 2
    3. 向左移动: 0不能在第一列, 即x%3 != 0
    4. 向右移动: 0不能在第三列, 即x%3 != 2
  • 对于已知起点和终点的问题, 我们可以采用双向bfs的思想, 即分别对起点和终点进行bfs, 用vis[]记录搜索状态, 将被起点搜索到的状态对应的vis[] += 1, 被终点搜索到的状态vis[] += 2. 这样一来, 倘若一个状态对应的vis[] ==3, 说明该状态被起点和终点同时搜索到了. 从而, 最少步数就等于起点到达该状态所用步数, 加上终点到达该状态所用步数.

算法自学__负权环问题

Posted on 2022-07-23 Edited on 2022-08-17 In algorithm

参考资料:

  1. https://www.jianshu.com/p/b876fe9b2338
  2. https://www.cnblogs.com/ztxcsl/p/13446263.html

问题引入

最短路中不能包括负权环, 因为每多经过一次负权环, 路径的权值都会进一步减小, 判断图中是否存在负权环, 我们一般采用Bellman-Ford算法和SPFA算法.

Bellman-Ford算法

松弛操作

我们以w(u,v)表示从顶点u到顶点v的边的权值, 以d[v]表示当前从起点s到顶点v的最短路径权值, 则有d[v] = min(d[v],d[u]+w(u,v))
容易证明, 经过一轮完整松弛操作至少能找到一个点的最短路径(具体证明见参考资料).

算法思想

假设图中有n个顶点, 我们对图进行n-1次松弛操作, 然后再执行一次松弛操作.

  • 若图中不存在负权环, 则经过前n-1次松弛操作, 已经找到了所有顶点的最短路径, 因此第n次松弛操作中不应该出现d[]数组更新的情况.
  • 若途中存在负权环, 则在第n次松弛操作中, 依然会出现d[]更新的情况

SPFA算法

SPFA算法是用队列优化的Bellman-Ford算法.

算法思想

  • 当前轮能够松弛成功的点, 一定是与上一轮松弛成功的点相连的点
  • 我们首先将起点s入队.
  • 假设当前出队顶点为u, 我们遍历所有以u为起点的边, 进行一轮松弛操作, 将所有松弛成功的点入队.
  • 若队列为空, 则所有顶点的最短路已经找到; 若某个顶点的入队次数>=n, 则说明图中存在负权环

例1 炮兵阵地

所谓状态压缩, 就是用二进制的性质来表示状态.
比如说, 在本题中, 若某一行为HHPPHPPH, 则可以用二进制数11001001表示
若某一行为炮空炮炮空炮空空, 则可以用二进制数10110100表示

  • 定义状态dp[line][i][j]: line表示当前行的序号(实际上只需要0, 1, 2), i表示当前行的炮的排列, j表示上一行的炮的排列.
  • 定义数组sum[], 用以存储每一种炮的排列中包含的炮的总数
  • 状态转移: i为当前行的炮排列, j为上一行的炮排列, k为上上行的炮排列.
  • 我们在设计算法时, 对于i, j, k可以采用枚举法, 即遍历00...0 ~ 11...11的每一个取值, 此时显然i, j, k的组合并不一定符合题意. 我可以从如下角度判断:
      1. 炮排列是否满足地形条件: 若`i&map[line] == 1`则不符合, `j, k`情况类似
  2. 某一行中是否存在炮的左右距离过近的情况: 若`i&(i<<1) == 1`或`i&(i<<2) == 1`则不符合条件, `j, k`情况类似
  3. 当前行的炮排列是否与前两行冲突: 若`i&j == 1`或`i&k == 1`或`j&k == 1`则不符合条件.

例3 愤怒的小鸟

题目分析

  • 设抛物线的表达式为y=ax^2+bx, 待定系数a和b可通过两组x, y来确定.
  • 我们任选两只小鸟i和j, 通过两只小鸟的坐标求出对应抛物线的a, b, 注意: a应该小于0, 然后我们遍历所有小鸟, 找出所有在这条抛物线上的小鸟. 由此, 我们得到了一个压缩的二进制数, 记录在line[i][j]中. 我们可以想象到, 在某些特殊的题目设定下, 有些小鸟可能不能与任何其他小鸟构成抛物线.
