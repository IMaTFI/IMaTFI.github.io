2 I/O设备的概念和分类

p262 8-1 什么是“设备独立性”？引入这一概念有什么好处？
答：设备独立性指，用户在程序中使用的设备与实际使用的设备无关，也就是在用户程序中仅使用逻辑设备名。引入设备独立性的好处：方便用户使用、改善设备利用率、提高系统的可扩展性和可适应性。

p262 8-3 什么是设备控制块？它主要应包含什么内容？简述其作用。
答：系统为每一台设备都配置了一个用来记录设备的硬件特性、连接和使用情况的一组数据，称为设备控制块。主要内容：设备名、设备属性、命令转换表（命令名、例程地址）。

p262 8-7 对I/O设备分配的一般策略是什么？
答：独享分配、共享分配、虚拟分配。

p262 8-8 什么是独占设备？对独占设备应如何分配？
答：独占设备即让一个作业在整个运行期间独占使用的设备。独占设备通常采用独享分配，在一个作业执行前，将它所要使用的设备分配给它；当它结束时，将分配给它的这类设备回收。

p262 8-9 什么是共享设备？对共享设备应如何分配？
答：共享设备即由多个作业、进程共同使用的设备。共享设备一般采用共享分配，在进程提出资源申请时，将所要使用的设备分配给它；进程使用完毕后，立即将设备收回。

p262 8-10 什么是虚拟设备技术？什么是虚拟设备？如何进行虚拟分配？
答：虚拟技术是在一类物理设备上模拟另一类物理设备的技术，是将独占设备转化为共享设备的技术。通常把用来代替独占性设备的那部分外存空间称为虚拟设备。虚拟分配的方法为：当进程需要与独占设备交换信息时，系统将分配磁盘空间，并建立相应的数据结构。

5 SPOOLing技术

p262 8-11 什么是SPOOLING系统？什么是预输入？什么是缓输出？
答：利用通道和中断技术，在主机控制之下，由通道完成输入输出工作。系统提供一个软件系统 (包括预输入程序、缓输出程序、井管理程序、预输入表、缓输出表)。它提供输入收存和输出发送的功能，使外部设备可以并行操作。这一软件系统称为SPOOLING系统。预输入：在应用程序需要数据前，OS已将数据预先输入到辅存输入井存放，当应用程序需要数据时，可直接从辅存中读入主存。缓输出：当应用程序执行时，将输出数据写入辅存输出井中，当应用程序执行完毕时（或需要数据时），由OS将数据输出。

1 文件管理概述

一个文件有哪些属性？

  • 文件名：由创建文件的用户指定，同一目录下不允许有重名文件。
  • 标识符：操作系统区分文件的内部名称。
  • 类型、大小、创建时间、保护信息等。

文件内部的逻辑结构：

  • 流式文件：无结构文件，由一些二进制或字符流组成。
  • 记录式文件：有结构文件，由一系列记录组成。

2 文件的逻辑结构

记录式文件：由一组相似的记录组成，每条记录包含若干数据项，每条记录有一个数据项可作为关键字。根据各条记录的长度，可分为定长记录和可变长记录两种。

单级目录结构：整个系统中只建立一个文件目录。在多用户环境中易出现命名冲突问题。

4 文件的物理结构

磁盘中的存储单元也会被分为一个个磁盘块，在很多操作系统中，磁盘块大小与内存块、页面大小相同。

4.1 连续分配方式

要求：每个文件在磁盘上占有一组连续的块。

优点：支持随机访问；顺序读/写速度最快（磁头移动距离短）。
缺点：不利于文件扩展；物理空间利用效率低，产生难以处理的磁盘碎片。

4.2 链接分配方式

隐式链接：

优点：磁盘利用率高，没有磁盘碎片；方便文件扩展。
缺点：只支持顺序访问，不支持随机访问。

显式链接：

优点：由于FAT表常驻内存，当我们想要访问某个逻辑块时，可以通过查表的方式找到其对应的物理块，所以显示链接是支持随机访问的；不会产生磁盘碎片；方便文件扩展。
缺点：FAT表需要占用一定的存储空间。

4.3 索引分配方式

优点：支持随机访问；方便文件扩展。
缺点：索引表占用存储空间。

如果一个文件的索引表的大小超过一个磁盘块的大小，应该如何处理？

各级索引表最大不能超过一个物理块。

数据库——数据库安全性

Posted on 2023-02-08

问题提出：

  • 数据库的一大特点是数据可以共享，但不是无条件的共享。
  • 数据库的数据共享在DBMS的严格控制之下，只允许有合法使用权限的用户访问相应的数据。

4.2 数据库安全性控制（P137）

4.2.1 用户身份鉴别（P138）

用户身份鉴别是系统提供的最外层安全保护措施。

基本方法：

  • 系统提供一定的方式让用户标识自己的名字或身份；
  • 系统内部记录着所有合法用户的标识；
  • 每次用户要求进入系统时，由系统核对用户提供的身份标识；
  • 通过鉴定后才提供机器使用权。
  • 用户标识和鉴定可以重复多次

4.2.2 存取控制（P140）

存取控制机制包括定义存取权限和检查存取权限两部分，二者共同构成DBMS的存取控制子系统。

  • 定义用户权限：在数据库系统中，为了保证用户只能访问他有权存取的数据，必须预先对每个用户定义存取权限。
  • 合法权限检查：对于通过鉴定获得上机权的用户（即合法用户），系统根据他的存取权限定义对他的各种操作请求进行控制，确保他只执行合法操作。

常用存取控制方法：

  • 自主存取控制：
    • 同一用户对于不同的数据对象有不同的存取权限
    • 不同的用户对同一对象也有不同的权限
    • 用户还可将其拥有的存取权限转授给其他用户
  • 强制存取控制：
    • 每一个数据对象被标以一定的密级
    • 每一个用户也被授予某一个级别的许可证
    • 对于任意一个对象，只有具有合法许可证的用户才可以存取

4.2.3 自主存取控制方法（P140）

自主存取控制（DAC）。

存取权限由两个要素组成：数据对象、操作类型。

在数据库系统中，定义存取权限称为授权。

优点：

  • 能够通过授权机制有效地控制其他用户对敏感数据的存取。

缺点：

  • 可能存在数据的“无意泄露”。原因：数据本身无安全性标记。解决：实施强制存取控制策略。

4.2.6 强制存取控制方法（P147）

强制存取控制（MAC）。

在MAC中，全部实体被分为主体和客体两大类：

  • 主体：系统中的活动实体
    • DBMS所管理的实际用户
    • 代表用户的各进程
  • 客体：系统中的被动实体，是受主体操纵的
    • 文件
    • 基表
    • 索引
    • 视图

敏感度标记：

  • 绝密（Top Secret）
  • 机密（Secret）
  • 可信（Confidential）
  • 公开（Public）

主体的敏感度标记称为许可证级别；客体的敏感度标记称为密级。
MAC机制就是通过对比主体和客体的敏感度，最终确定主体是否能够存取客体。

规则：

  • 仅当主体的许可证级别大于或等于客体的密级时，该主体才能读取相应的客体；
  • 仅当主体的许可证级别小于或等于客体的密级时，该主体才能写相应的客体。

第二条规则为了防止禁止了拥有高许可证级别的主体更新低密级的数据对象。例如，某TS级主体读取TS级数据后，将其写入P级数据中，造成了数据泄露。

MAC控制的特点：

  • MAC是对数据本身进行密级标记
  • 无论数据如何复制，标记与数据是一个不可分的整体
  • 只有符合密级标记要求的用户才可以操纵数据，从而提供了更高级别的安全性

数据库——数据库完整性

Posted on 2023-02-08

5.1 实体完整性（P158）

5.1.1 定义实体完整性（P158）

关系模型的实体完整性在 CREATE TABLE 中用 PRIMARY KEY 定义。

5.1.2 实体完整性检查和违约处理（P159）

当用户插入一条记录或对主码列进行更新时，数据库管理系统会进行检查：

  1. 检查主码值是否唯一，如果不唯一则拒绝插入或修改。
  2. 检查主码的各属性是否为空，只要有一个为空则拒绝插入或修改。

检查主码是否唯一的方式的一种方法是进行全表扫描，但该方法过于耗时，所以数据库管理系统一般会在主码上建立索引。

5.2 参照完整性（P160）

5.2.1 参照完整性（P160）

关系模型的参照完整性在 CREATE TABLE 中用 FOREIGN KEY 定义哪些列为外码，用 REFERENCES 指明这些外码参照那些表的主码。

5.2.2 参照完整性检查和违约处理（P160）

对参照表和被参照表进行删除、修改操作都可能破坏参照完整性。

当上述不一致发生时，系统可以采用如下策略加以处理：

  1. 拒绝执行（NO ACTION）（默认）。
  2. 级联操作（CASCADE）：当删除或修改被参照表的一个元组导致与参照表不一致时，删除或修改参照表所有导致不一致的元组。
  3. 设置为空值：当删除或修改被参照表的一个元组导致与参照表不一致时，将参照表所有导致不一致的元组的对应属性设置为空值。

外码能够取空值，要根据该外码在关系中是否同时作为主码来决定。所以对抑郁参照完整性，还应定义外码列是否允许空值。

5.3 用户定义完整性（P163）

5.3.1 属性上的约束条件（P163）

在使用 CREATE TABLE 定义属性时，可以同时对属性加以限制：

  • 列值非空（NOT NULL）
  • 列值唯一（UNIQUE）
  • 检查列值是否满足一个条件表达式（CHECK 短语）

向表中插入元组或修改属性的值时，如果不满约束条件，则直接拒绝执行。

5.3.2 元组上的约束条件（P164）

插入元组或修改属性的值时，关系数据库管理系统检查元组上的约束条件是否被满足，如果不满足则操作被拒绝执行。

5.4 完整性约束命名子句（P165）

SQL在 CREATE TABLE 语句中提供了完整性约束命名子句 CONSTRAINT。

格式：CONSTRAINT <完整性约束条件名> <完整性约束条件>

完整约束条件包括：NOT NULL、UNIQUE、PRIMARY KEY、FOREIGN KEY、CHECK。

可以使用 ALTER TABLE 修改表中的完整性约束限制。

5.6 断言（P167）

SQL中，可以使用 CREATE ASSERTION 语句，通过声明性断言来指定更具一般性的约束。

断言创建以后，任何对断言中所涉及的关系的操作都会触发关系数据库管理系统对断言的检查，任何使断言不为真值的操作都会被拒绝执行。

创建断言的语句格式：CREATE ASSERTION <断言名> <CHECK 子句>
每个断言都被赋予一个名字，<CHECK 子句>中的约束条件与WHERE子句的条件表达式类似。

删除断言的语句格式：DROP ASSERTION <断言名>

如果断言很复杂，则系统在检测和维护断言的开销较高，这是在使用断言时应该注意的。

5.7 触发器（P168）

触发器（Trigger）是用户定义在关系表上的一类由事件驱动的特殊过程。

触发器保存在数据库服务器中，任何用户对表的增、删、改操作均由服务器自动激活相应的触发器。

1 进程的定义

p130 4-1 试解释下列名词：程序的顺序执行、程序的并发执行。
答：一个计算的若干操作必须按照严格的先后次序顺序地执行，这个计算过程就是程序的顺序执行过程。程序的顺序执行具有顺序性、封闭性和可再现性。若干个程序段同时在系统中运行，这些程序段的执行在时间上是重叠的，一个程序段的执行尚未结束，另一个程序段的执行已经开始，即使这种重叠是很小的一部分，也称这几个程序段是并发执行的。程序的并发执行不具有封闭性和可再现性。

进程是程序的一次执行过程，是系统进行资源分配调度的一个基本单位。进程由程序段、数据段和PCB三部分构成。

p131 4-3 什么是进程？进程和程序的主要区别是什么？
答：进程是一个具有独立功能的程序关于某个数据集合的一次运行活动。程序是静态的概念，进程是动态的概念；进程是一个独立运行的活动单位；进程是竞争系统资源的基本单位；一个程序可以对应多个进程，一个进程至少包含一个程序。

p132 什么是进程控制块？
答：描述进程与其他进程、系统资源的关系以及进程在各个不同时期所处的状态的数据结构，称为进程控制块 (process control block，PCB)。

5 线程

线程可以理解为轻量级进程。

p133 什么是线程？线程和进程的区别是什么？
答：线程也称为轻量级进程，它是比进程更小的活动单位，它是进程中的一个执行路径，一个进程可能有多个执行路径，即线程。主要区别：(1) 线程是进程的一个组成部分，一个进程可以有多个线程，而且至少有一个可执行的线程。(2) 进程是资源分配的基本单位，它拥有自己的地址空间和各种资源；线程是处理机调度的基本单位，它只能和其他线程共享进程的资源，而本身并没有任何资源。(3) 进程的多个线程都在进程的地址空间内活动。这样，在以线程为单位进行处理机调度和切换时，切换时间较短；而以进程为单位进行处理机调度和切换时，由于涉及到资源转移及现场保护等问题，将导致切换时间变长和资源利用率下降。(4) 线程和进程一样，都有自己的状态和相应的同步机制，但是，由于线程没有自己单独的程序和数据空间，因而不能像进程那样将程序和数据交换到外存去。(5) 进程的调度和控制大多由操作系统的内核完成，而线程的控制既可以由操作系统内核完成，也可以由用户控制完成。


同一进程的不同线程共享进程的资源。
同一进程的不同线程之间的通信无需操作系统的干预。

6 进程的同步、互斥

同步：亦称直接制约关系，强调进程间的工作次序。

临界资源：一个时间段内只允许一个进程使用的资源。

对临界资源的访问必须互斥进行。

p132 4-16 什么是进程的互斥？什么是进程的同步？同步和互斥这两个概念有什么联系和区别？
答：互斥：某些资源在一个时间段内只允许一个进程访问。同步：并发进程在一些关键点上需要相互等待与互通消息。联系：同步和互斥都描述了并发进程间的约束关系。区别：同步强调进程间的先后次序，互斥强调一个时间段内只允许一个进程使用临界资源。

死锁：各进程互相等待对方手里的资源，导致各进程阻塞。
饥饿：某进程长期得不到想要的资源，无法向前推进。

p155 5-8 什么是死锁？
答：在两个或多个并发进程中，如果每个进程持有某种资源而又都等待着别的进程释放它或它们现在保持着的资源，否则就不能向前推进。此时，称这一组进程产生了死锁。

操作系统——主存管理

Posted on 2023-02-01 Edited on 2023-02-15

1 内存管理的概念

操作系统负责：

  • 内存空间的分配与回收。
  • 内存空间的扩充（实现虚拟性）。
  • 地址转换：逻辑地址 → 物理地址
  • 存储保护：保证各进程在自己的内存空间内运行，不会越界访问。

p220 7-2 什么是逻辑地址？什么是物理地址？为什么要进行二者的转换？
答：物理地址：计算机主存单元的真实地址。逻辑地址：用户的程序地址。支持多个用户程序共享内存、方便用户使用。

地址映射的三种方式：

  1. 绝对装入：编译时产生绝对地址。
  2. 静态地址映射：程序装入时进行地址变换。
  3. 动态地址映射：运行时进行地址转换，进程的起始物理地址保存在PCB中，需设置重定位寄存器。

p221 7-3 什么是静态地址重定位？它需要什么支持？
答：静态地址映射：在程序装入过程中进行地址变换。需要重定位装入程序（软件）支持。

p221 7-4 什么是动态地址重定位？它需要什么支持？
答：动态地址映射：在程序运行时进行地址变换。需要重定位寄存器（硬件）支持。

存储保护有两种实现方式：

  1. 设置上、下限寄存器，存放进程的上、下限物理地址，进程的指令要访问某个地址时，CPU检查是否越界。
  2. 采用重定位寄存器（基址寄存器）和界地址寄存器（限长寄存器）进行越界检查。重定位寄存器存储进程的起始物理地址，界地址寄存器存储进程的最大逻辑地址。

2 覆盖与交换

2.1 覆盖

覆盖技术用来解决程序大小超过物理内存总和的问题。

覆盖技术的思想 ：将程序分为多个段，常用的段常驻内存，不常用的段在需要时调入内存。

内存分为一个固定区和若干覆盖区。
需要常驻的段放在固定区，调入后不再调出（除非运行结束）；不常用的段放在覆盖区，需要时调入内存，不用时调出内存。

同一层的程序段不能同时访问，所以可以共享一片覆盖区。

缺点：必须由程序员声明覆盖结构（如上图），增加了用户的编程负担。

2.2 交换

交换技术的设计思想：内存紧张时，系统将内存中某些进程换出到外存，把外存中某些已具备运行条件的进程换入到内存。

在具有交换功能的操作系统中，磁盘通常被分为文件区和对换区。文件区采用离散分配方式，对换区采用连续分配方式。对换区的I/O速度比文件区更快。

交换通常在内存不足时进行，内存宽裕时便暂停。

3 连续分配管理方式

连续分配：为用户进程分配的必须是一个连续的内存空间。

3.1 单一连续分配

内存被分为系统区和用户区，系统去通常位于内存的低地址部分。内存中只能有一道用户程序，用户程序独占用户区。

优点：实现简单，无外部碎片

缺点：只能用于单用户、单任务的操作系统中，有内部碎片，存储器利用效率低。

内部碎片：分配给某进程的内存区域中，如果有些区域没有用上，就是内部碎片。

3.2 固定分区分配

将用户空间划分为若干个固定大小的分区（分区的大小可以完全相同，也可以不同，但一定是固定的），在每个分区只能装入一个作业。

操作系统需要建立一个数据结构——分区说明表，每个表项对一个分区，通常按照分区大小排列。

优点：实现简单，无外部碎片。

缺点：当用户程序过大时，不得不使用覆盖技术来解决；会产生内部碎片。

4 离散分配管理方法

4.1 基本分页存储

将内存空间分为一个个大小相等的页框（如4KB），每个页框有一个页框号，从0开始。将用户进程的地址空间也分为与页框大小相等的一个个页面，每个页面有一个页号，从0开始。

如何实现地址转换？

  1. 计算逻辑地址对应的页号。
    页号 = 逻辑地址 / 页面长度（取结果的整数部分）
  2. 获取该页面在内存中的起始物理地址。
  3. 算出逻辑地址在页面内的偏移量。
    页内偏移量 = 逻辑地址 % 页面长度
  4. 物理地址 = 页面始址 + 页内偏移量。

在计算机中，一般将页面大小设置为 字节，所以如果给定一个二进制逻辑地址，后 位即为页内偏移量，前面的位为页号。

4.1.1 页表

  • 一个进程对应一张页表。
  • 每个页表项由页号和块号组成
  • 每个页表项的长度是相同的，页号是隐含的（相当于数组下标）。

容易混淆的几个名词：页表长度、页表项长度（实际上就是页框号需要用多少字节存储，但由于页表也要存储在内存中，所以页表项长度最好能整除页表长度）、页面大小。

由于页表也存储在内存中，所以每次取数据实际要访存两次。为此，引入快表（联想寄存器TLB）缓存经常访问的若干页表项

4.1.2 多级页表

单级页表的问题：

  • 页表必须连续存放，当页表很大时，需要占用很多个连续的页框。
  • 没有必要让整个页表常驻内存。

思想：将页表拆分成多个分组，使每个页框恰好可以放入一个分组，然后再为离散的页表再建立一张页表，称为页目录表。

需要注意的细节：

  • 各级页表的大小均不能超过一个页框，所以有些情况下可能要建立多级页表

答：页式系统的地址结构是一维的，段式系统的地址结构是二维的。

p222 7-23 页式系统的分页和段式存储的分段有什么区别？
答：分页是信息的物理划分，页的大小是固定的，用户不可见，页内偏移的溢出会自动加入到页号中。分段是信息的逻辑划分，段长是可变的，用户可见，段内偏移溢出将产生越界中断。

5 虚拟内存

5.1 虚拟内存的基本概念

传统存储方式的缺点：

  • 一次性：作业必须一次性全部装入内存后才能开始运行。
  • 驻留性：一旦作业被装入内存中，就会一直驻留在内存中。

基于局部性原理，在程序装入时，可以将程序很快用到的部分载入内存，将暂时用不到的部分留在外存，在需要时由操作系统调入内存。

虚拟内存的最大容量由CPU寻址范围确定，实际容量 = min（内外存容量之和，CPU寻址范围）。

虚拟内存的三个特征：

  • 多次性：允许作业分成多次调入内存。
  • 对换性：允许在作业运行的过程中，将作业换入、换出。
  • 虚拟性：从逻辑上扩充了内存的容量。

p221 7-15 什么是虚拟存储器？在页式系统中如何实现虚拟存储？
答：由操作系统和硬件相配合来完成主存和辅存之间的信息的动态调度，这样的计算机系统好像为用户提供了一个存储容量比实际主存大得多的存储器，这个存储器称为虚拟存储器。虚拟存储器的核心：逻辑地址与物理地址分开、存储空间和虚地址空间分开、提供地址变换机构。

5.2 请求分页管理


当要访问的页面不在内存时，会产生缺页中断，缺页的进程会被放入阻塞队列，调页完成后再将其唤醒，放回就绪队列。

缺页中断属于内中断，一个指令可能引起多次缺页中断。

如果内存中有空闲块，则将缺失的页面载入块中；如果没有相应的空闲块，则通过页面置换算法将一个页面淘汰，如果淘汰页面在内存期间被修改过，还要将其写回外存。（注意要及时修改页表）

页面调入内存后，需要同时修改快表和慢表。

5.3 页面置换算法

p221 7-17 什么是系统的抖动？它有什么危害？
答：主存与辅存之间频繁的页面置换现象称为颠簸。这将导致系统效率急剧下降。

p221 7-18 什么是置换算法？在页式系统中常用的置换算法是什么？
答：当要索取一页面并送入主存时，必须将该程序中的某一页面淘汰掉。用来选择淘汰哪一页的规则就叫做置换算法。常用的置换算法有：先进先出置换算法、最近最久未使用置换算法、时钟置换算法。

1 操作系统的概念、功能和目标

操作系统的定义：

  • 负责管理协调硬件、软件等计算机资源
  • 为上层的应用程序、用户提供简单易用的服务
  • 计算机中最基本的系统软件

作为系统资源的管理者：

  • 功能：处理机管理、存储器管理、文件管理、设备管理
  • 目标：安全高效

作为用户和计算机硬件之间的接口：

  • 命令接口：允许用户直接使用（如：命令行中的指令）
  • 程序接口：允许用户通过程序间接使用
  • 图形用户界面

作为最接近硬件的层次：

  • 实现对硬件机器的扩展

p33 1-1 存储程序式计算机的特点：指令流驱动计算、集中顺序过程控制（集中：CPU集中管理、顺序：程序计数器控制、过程：模拟人手工操作）。

p34 1-7 什么是操作系统？操作系统的特征是什么？
操作系统（OS）是一个大型的软件系统，它负责计算机系统软、硬件资源的分配，控制和协调并发活动，提供给用户和其他软件方便的接口和环境，使用户获得良好的工作环境。特点：并发、共享、虚拟、不确定。

2 操作系统的特征

  • 并发：宏观上同时进行，微观上交替进行
  • 共享：系统中的资源可供多个并发进程共同使用（互斥共享、同时共享）
  • 虚拟：把一个物理上的实体变为若干逻辑上的对应物
  • 异步（不确定）：多个程序并发执行时，每个进程的执行是“走走停停”的，推进速度是不可预知的

p53 什么是操作系统虚拟机？
答：配置了操作系统的裸机。虚拟机没有中断；虚拟机为每个任务提供了一个虚拟处理器；虚拟机提供了更强的指令系统。

p53 在设计操作系统时，可以考虑的结构组织有哪几种？
答：单内核、微内核、伴生内核。

3 操作系统用的发展和分类

3.1 手工操作阶段

3.2 单道批处理系统

3.3 多道批处理系统

P33 1-5 什么是多道程序设计技术？试分析多道程序设计技术的特征？
答：多道程序设计技术：在计算机主存中同时存放几道相互独立的程序。这些程序在管理程序控制之下，相互穿插地运行。当某道程序因某种原因不能继续运行下去时(如等待外部设备传输数据)，管理程序便将另一道程序投入运行。特点：多道 （内存中存放多个相互独立的程序）、宏观上并行 （相互穿插运行）、微观上串行 （CPU上永远只能执行单个程序）。

3.4 分时操作系统

P33 1-2 批处理系统和分时系统各具有什么特点？为什么分时系统响应比较快？
答：批处理系统的特点：程序包中的上一个程序执行完毕后，下一个程序自动投入运行，程序包中的程序执行切换无需人工干预。分时系统的特点：联机交互，独占使用，响应时间快

P34 1-6 什么是分时技术？
答：所谓分时技术，是把处理机时间划分成很短的时间片(如几百毫秒)轮流地分配给各个应用程序使用，如果某个程序在分配的时间片用完之前计算还未完成，该程序就暂停执行，等待下一次获得时间片后再继续计算。

3.5 实时操作系统

P33 1-3 实时系统和分时系统的本质区别？
答：实时系统强调可预测性，即在规定的时间内完成事务的处理。

4 操作系统的运行机制与体系结构

指令分为特权指令和非特权指令。特权指令不允许用户程序执行。

处理器有两种状态：用户态（目态）和核心态（管态）。通过程序状态字寄存器PSW中的标志位来标识当前处理器的状态。处于用户态的CPU只能执行非特权指令。

程序分为内核程序和应用程序。内核程序运行在核心态，应用程序运行在用户态。

p53 2-3 什么是处理机的态？为什么要区分处理机的态？
答：处理机所处的状态、正在处理哪类程序；保护操作系统。

p53 2-4 什么是管态？什么是用户态？
答：管态：操作系统的管理程序执行时机器所处的状态，又称处理机的特权态。在此状态下处理机可使用全部指令（包括一组特权指令）、使用全部系统资源（包括整个存储区域）。用户态：用户程序执行时机器所处的状态称为用户态。在此状态下禁止使用特权指令，不能直接取用资源与改变机器状态，并且只允许用户程序访问自己的存储区域。

5 中断和异常

发生中断意味着需要操作系统介入，开展管理工作。

中断发生后，CPU立即进入核心态，当前进程暂缓执行。中断时用户态转换到核心态的唯一途径，中断可以使操作系统得到计算机的控制权。

p53 2-5 什么是中断？在计算机系统中为什么要引入中断？
答：所谓中断是指某个事件 (例如电源掉电、定点加法溢出或I/O传输结束等)发生时，系统中止现行程序的运行、引出处理事件程序对该事件进行处理，处理完毕后返回断点继续执行的过程。应用程序（用户态）的执行时常需要操作系统（内核态）的“帮助”。中断的本质是受保护的状态转换，转换的过程中不允许存在让用户程序干预的可能。

p53 2-6 按中断的功能分，中断有哪几种类型？
答：输入输出中断、外中断、机器故障中断、程序性中断、访管中断。

p53 2-7 什么是强迫性中断？什么是自愿中断？试举例说明。
答：强迫性中断不是当前程序所期待的，而自愿中断是当前程序所期待的。访管中断属于自愿中断类型。

p53 2-8 中断和俘获有什么不同？
答：中断源于CPU外部，俘获源于CPU内部。

p53 2-9 什么是中断响应？其实质是什么？
答：中断响应指CPU发现已有中断请求，中止当前程序执行，并自动引出中断处理程序的过程。其实质是交换用户程序和中断处理程序的指令执行地址和处理机状态。

p53 2-10 什么是程序状态字？在微机中其一般由哪两部分组成？
答：PSW是反应程序执行时机器所处的现行状态的代码，包括：指令地址、指令执行情况、处理机状态、应屏蔽的中断等。PSW的主要作用是实现程序状态的保护和恢复。在微机中，一般由指令计数器和处理机状态寄存器组成。

p53 2-11 什么是向量中断？什么是中断向量？
答：向量中断：当中断发生时，由中断源自己引导处理机进入中断服务程序的中断过程称为向量中断。中断向量：每类中断类型都有自己的中断向量，包含：中断服务例程的入口地址和程序状态字。

p53 2-12 软件中断的处理过程？
答：保护现场和传递参数；执行相应的中断服务例程；恢复和退出中断。

6 系统调用

系统调用的过程：传递系统调用所需的参数 -> 执行陷入指令 -> 执行相应系统调用的服务程序 -> 返回用户程序。

陷入指令在用户态执行，执行陷入指令后立即引发一个内中断，使CPU进入核心态。
发出系统调用请求在用户态，对系统调用进行处理在核心态。
陷入指令是唯一一个只能在用户态执行，不能在核心态执行的指令。

p73 3-1 什么是系统生成？
答：所谓系统生成，就是指为了满足物理设备的约束和需要的系统功能，通过组装一批模块来产生一个清晰的、使用方便的操作系统的过程。简单地说就是系统的安装，即将OS安装到启动装置（可以是磁盘，也可以是tftp服务器的某个目录）

p73 3-2 系统引导的主要任务是什么？
答：将操作系统的必要部分装入主存并对系统进行初始化工作，最终使系统处于命令接收状态。主要有独立引导方式和辅助下装方式两种。

p73 3-3 处理应用程序分哪几个步骤？这几个步骤之间是什么关系？
答：步骤：编辑、编译、链接、运行。这4个步骤相互关联、顺序执行。

p73 3-4 静态链接和动态链接有什么区别？
答：静态链接将所有外部函数的调用都连接到目标文件中形成一个完整的主存映像，动态链接是将连接这一工作延迟到程序运行时进行。

p73 3-5 用户与操作系统的接口是什么？
操作系统的用户界面 (或称接口) 是操作系统提供给用户与计算机打交道的外部机制。用户能够借助这种机制和系统提供的手段来控制用户所在的系统。

p73 3-8 什么是系统功能调用？对操作系统的服务请求与一般的子程序调用有什么区别？
答：系统调用是用户在程序一级请求操作系统服务的一种手段，它是带有一定功能号的访管指令。其功能是由操作系统中的程序完成的，即由软件方法实现的。系统调用代码属于OS（驻留在内存），一般库函数由开发软件提供，由编译工具链入用户程序。系统调用代码的执行引起CPU状态的变化：用户态核心态，一般库函数的执行不会引起CPU状态的变化：用户态。系统调用减小了用户程序大小。

p73 3-9 为了实现系统调用，操作系统设计者应该做哪些工作？
答：（1）编写实现各种子功能的例行子程序，并对其进行编号。（2）构造例行子程序入口地址表，各个例行子程序的入口地址按其编号存入表中。（3）编写访管中断处理程序，该程序的功能为：保护现场；取系统功能调用号i，根据例行子程序入口地址表和i值跳转执行相应例程。

p73 3-10 简述系统功能调用的执行过程。
答：用户程序执行到系统调用命令时，会发生中断，处理器由用户态变为核心态，操作系统的访管中断程序得到控制权。它根据系统调用的功能号，根据例行子程序入口地址表跳转到相应的例程去执行，完成用户要求的服务功能后，退出中断，返回用户程序断点继续执行。
